## 已完成内容

### 1. 元数据权限持久化

- 已将权限信息作为元数据的一部分持久化保存。
- 文件默认权限为 `0644`，目录默认权限为 `0755`。
- `stat` / `getattr` 返回的权限值与元数据保持一致。

### 2. 创建路径权限写入

- 创建文件时写入默认文件权限。
- 创建目录时写入默认目录权限。
- FUSE 创建路径会根据传入 mode 更新权限，并在写入前过滤特殊位。

### 3. chmod 能力

- 已提供 `chmod` 链路，并将权限更新持久化到元数据层。
- `chmod` 后再次 `stat`，返回结果与修改后的权限一致。
- 对超过普通权限范围的位进行掩码处理，仅保留 `0777`。

### 4. FUSE setattr 映射

- FUSE 层已将 `FATTR_MODE` / `setattr` 的 mode 更新映射到权限修改逻辑。
- `chown` 等未实现复杂属性在 FUSE 层返回 `ENOSYS`。

### 5. 安全限制

- `setuid`、`setgid`、`sticky` 位在支持的 mode 更新入口均被直接清除。
- 已验证 `chmod 4755` 不会保留特殊位。
- 已修复创建路径上的特殊位问题，`mkdir -m 1777` 不再保留 `sticky` 位。

### 6. 错误处理

- 不存在路径的 `chmod` 返回 `ENOENT`。
- 非法模式在写入前进行位清理后继续执行。

### 7. 并发与原子性

- 权限更新沿用现有元数据后端的事务或原子更新机制。
- 数据库后端通过事务保证更新一致性。
- etcd / Redis 路径沿用各自已有的原子更新模型。

### 8. 文档补充

- 已补充 `docs/permissions.md`，说明当前支持范围、限制项与行为边界。

## 测试结论

### 单元测试

- 权限相关单元测试已全部通过。
- 当前分支上的所有单元测试已通过。

### Linux FUSE 手测

手测脚本：`project/slayerfs/tests/scripts/manual_fuse_permissions.sh`

关键检查项均已通过：

- 默认文件权限为 `644`。
- 默认目录权限为 `755`。
- `chmod` 后权限立即生效。
- 重挂载后权限仍然保持一致。
- `chmod 4755` 最终返回 `755`。
- 缺失路径 `chmod` 返回 `ENOENT`。
- `mkdir -m 1777` 最终返回 `777`。
- `chown` 返回 `ENOSYS` / `Function not implemented`。
